# Ejercicio Nro: 3

## Enunciado

### ¿Qué ventajas y desventajas tienen los diagramas UML?

Ayuda: Se puede relacionar con los planos de la Ingeniería tradicional.

## Resolución

Los diagramas UML son una herramienta muy útil para representar la estructura y el funcionamiento de un sistema antes de comenzar a programarlo. Permiten organizar las ideas, documentar el proyecto y facilitar la comunicación entre los integrantes del equipo.

### Ventajas

- Ayudan a comprender el funcionamiento del sistema antes de implementarlo.
- Facilitan la comunicación entre desarrolladores, analistas y clientes.
- Permiten detectar errores o problemas de diseño en etapas tempranas.
- Sirven como documentación para el mantenimiento y futuras modificaciones del software.

### Desventajas

- Requieren tiempo para elaborarlos y mantenerlos actualizados.
- Si el proyecto cambia con frecuencia, los diagramas pueden quedar desactualizados rápidamente.
- No representan todos los detalles del funcionamiento real del software.
- Un buen diagrama no garantiza que el software final sea de calidad.

En mi opinión, los diagramas UML son una buena herramienta para planificar y comunicar el diseño del sistema, pero no deben reemplazar el desarrollo y la interacción con el cliente.
