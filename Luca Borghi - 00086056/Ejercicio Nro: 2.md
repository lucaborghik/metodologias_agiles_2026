# Ejercicio Nro: 2

## Enunciado

### ¿Qué es para vosotros el desarrollo de software?

Vamos a contestar entre todos algunas preguntas para conocer vuestras opiniones y experiencias:

- ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
- ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
- ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
- ¿Cuándo se ha desarrollado un buen software?

## Resolución

Para mí, el desarrollo de software es el proceso de analizar un problema, diseñar una solución, programarla, probarla y mantenerla para que realmente sea útil para las personas que la van a utilizar. No se trata solamente de escribir código, sino de crear un producto que resuelva una necesidad.

Durante la carrera participé en distintos proyectos académicos, como el desarrollo de aplicaciones en C#, Python y JavaScript, además de bases de datos en SQL y un proyecto más completo llamado Nativo System, en el que fue necesario analizar los requerimientos, diseñar el sistema y desarrollar distintas funcionalidades.

El desarrollador tiene la responsabilidad de transformar los requerimientos del cliente en un software funcional. Para lograrlo, debe escribir código de calidad, realizar pruebas, corregir errores y trabajar en conjunto con el resto del equipo para cumplir los objetivos del proyecto.

Además del desarrollador, existen otros roles muy importantes, como el cliente o usuario, el Product Owner, el Scrum Master, los analistas funcionales, los testers y los diseñadores UX/UI. Cada uno aporta conocimientos diferentes que ayudan a desarrollar un mejor producto.

Considero que un software es bueno cuando resuelve correctamente el problema para el que fue creado, funciona de manera confiable, es fácil de usar, puede mantenerse y adaptarse a nuevos cambios, y cumple con las necesidades del usuario.
