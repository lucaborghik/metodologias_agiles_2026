# Ejercicio Nro: 15

## Enunciado

Generar un plan de trabajo basado en SCRUM para resolver la siguiente tarea:

**Objetivo:** El objetivo de este ejercicio es que los alumnos universitarios practiquen la creación de historias de usuario para un sistema informático de presupuesto de construcción de galpones, utilizando la metodología ágil.

## Resolución

# Sistema de Presupuesto para Construcción de Galpones

## Historia de Usuario 1

**Título:** Crear un presupuesto de construcción.

### Historia de Usuario

Como vendedor, quiero crear un presupuesto para un nuevo galpón para ofrecer al cliente un costo estimado de la obra.

### Criterios de aceptación

- El sistema debe permitir ingresar los datos del cliente.
- Debe permitir seleccionar el tipo de galpón.
- Debe calcular automáticamente el presupuesto.
- Debe guardar el presupuesto para futuras consultas.

---

## Historia de Usuario 2

**Título:** Seguimiento del presupuesto.

### Historia de Usuario

Como cliente, quiero consultar el estado de mi presupuesto para conocer el avance de mi solicitud.

### Criterios de aceptación

- El cliente debe poder visualizar el estado del presupuesto.
- Debe mostrarse la fecha de creación.
- Debe indicar si el presupuesto está pendiente, aprobado o rechazado.
- La información debe actualizarse cuando existan cambios.

---

## Historia de Usuario 3

**Título:** Generación de informes.

### Historia de Usuario

Como gerente, quiero generar informes de presupuestos para analizar las ventas y tomar decisiones.

### Criterios de aceptación

- El sistema debe generar informes por período.
- Debe mostrar la cantidad de presupuestos realizados.
- Debe incluir el monto total presupuestado.
- Debe permitir exportar el informe en formato PDF.

## Explicación

Estas historias de usuario describen tres funcionalidades principales del sistema: crear presupuestos, realizar el seguimiento de su estado y generar informes para la gestión. Cada historia está redactada desde la perspectiva del usuario y define criterios de aceptación que permiten verificar cuándo la funcionalidad puede considerarse terminada.
