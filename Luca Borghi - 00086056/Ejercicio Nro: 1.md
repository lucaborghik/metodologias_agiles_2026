# Ejercicio Nro: 1

## Enunciado

### ¿Qué sabéis de metodologías ágiles?

- ¿Conceptos aprendidos en otras asignaturas?
- ¿Estáis trabajando? ¿Qué metodología usan en la empresa?
- ¿Herramientas de integración y entrega continua?
- ¿Cuáles pensáis que son las características principales de un desarrollo ágil?

## Resolución

Durante la carrera aprendí que las metodologías ágiles son una forma de desarrollar software de manera iterativa e incremental, entregando pequeñas partes funcionales del sistema en lugar de esperar hasta el final del proyecto. Su objetivo es adaptarse rápidamente a los cambios y mantener una comunicación constante con el cliente.

Todavía no trabajo en una empresa de desarrollo de software, por lo que no tengo experiencia práctica con una metodología específica. Sin embargo, conozco metodologías como Scrum y Extreme Programming (XP), que son de las más utilizadas en proyectos ágiles.

También conozco algunas herramientas relacionadas con la integración y entrega continua, como Git y GitHub para el control de versiones, y GitHub Actions o Jenkins, que permiten automatizar pruebas y despliegues del software.

Considero que las principales características de un desarrollo ágil son:

- Trabajo en equipo y buena comunicación.
- Entregas frecuentes de software funcionando.
- Adaptación a cambios en los requisitos.
- Participación constante del cliente.
- Mejora continua durante todo el desarrollo.
