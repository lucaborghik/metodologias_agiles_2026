# Ejercicio Nro: 7

## Enunciado

Lee el paper **"No Silver Bullet"** de Fred Brooks y explica con un ejemplo cada una de las cuatro dificultades esenciales del desarrollo de software.

## Resolución

Según el paper de Fred Brooks, el desarrollo de software presenta cuatro dificultades esenciales. Un ejemplo de cada una es el siguiente:

### 1. Complejidad

Un sistema bancario que permite realizar transferencias, pagos, préstamos e inversiones tiene muchas funciones que deben interactuar entre sí. Cuantas más funcionalidades se agregan, más difícil es desarrollar, probar y mantener el sistema.

### 2. Conformidad

Una empresa desarrolla un sistema de facturación que debe cumplir con las normas fiscales y los requisitos establecidos por ARCA (ex AFIP). Aunque esas reglas sean complejas o cambien con el tiempo, el software debe adaptarse para cumplirlas correctamente.

### 3. Cambiabilidad

Una aplicación de ventas ya está terminada, pero el cliente solicita agregar una nueva forma de pago y modificar el proceso de compra. El equipo debe realizar cambios sin afectar el funcionamiento del resto del sistema.

### 4. Invisibilidad

A diferencia de una construcción, el software no puede observarse físicamente. Es difícil visualizar toda su estructura y funcionamiento, por lo que se utilizan diagramas, documentación y modelos para representar cómo está organizado el sistema.
